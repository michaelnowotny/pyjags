    Model¶

    The core PyJAGS API for model specification, compilation, and MCMC sampling.

    Model ¶

    High level representation of JAGS model.

    Attributes:

    Name Type Description
    chains int

    A number of chains in the model.
    Note

    In JAGS arrays are indexed from 1. On the other hand Python uses 0 based indexing. It is important to keep this in mind when providing data to JAGS and interpreting resulting samples. For example, what in JAGS would be x[4,2,7] in Python is x[3,1,6].

    Note

    The JAGS supports data sets where some of observations have no value. In PyJAGS those missing values are described using numpy MaskedArray. For example, to create a model with observations x[1] = 0.25, x[3] = 0.75, and observation x[2] missing, we would provide following data to Model constructor:

    {'x': np.ma.masked_array(data=[0.25, 0, 0.75], mask=[False, True, False])} {'x': masked_array(data = [0.25 -- 0.75], mask = [False True False], fill_value = 1e+20) }

    From JAGS version 4.0.0 it is also possible to monitor variables that are not completely defined in the description of the model, e.g., if y[i] is defined only for y[3], then y[1], and y[2] will have missing values for all iterations in all chains. Those missing values are also represented using numpy MaskedArray.

    variables property ¶

    Variable names used in the model.

    Returns:

    Type Description
    list[str]

    Names of all variables defined in the JAGS model.

    state property ¶

    Values of model parameters and model data for each chain.

    Returns:

    Type Description
    list[dict[str, ndarray | MaskedArray]]

    A list with one dictionary per chain. Each dictionary maps variable names to their current values, including both parameters and data.
    See Also

    parameters data

    parameters property ¶

    Values of model parameters for each chain.

    Includes the name of the random number generator as '.RNG.name' and its state as '.RNG.state'.

    Returns:

    Type Description
    list[dict[str, ndarray | MaskedArray]]

    A list with one dictionary per chain. Each dictionary maps parameter names to their current values.

    data property ¶

    Model data for the first chain.

    Includes data provided during model construction and data generated as part of the data block in the JAGS model.

    Returns:

    Type Description
    dict[str, ndarray | MaskedArray]

    Dictionary mapping variable names to their data values.

    Information about the samplers used for each node.

    Type Description
    list[list[str]]

    A list of sampler descriptions. Each inner list contains the node name and the sampler method assigned to it.

    __init__(code=None, data=None, init=None, chains=4, adapt=1000, file=None, encoding='utf-8', generate_data=True, progress_bar=True, refresh_seconds=None, threads=1, chains_per_thread=1, seed=None) ¶

    Create a JAGS model and run adaptation steps.

    Parameters:

    Name Type Description Default
    code str or bytes

    Code of the model to load. Model may be also provided with file keyword argument.

    		 None
    file str or path - like

    Path to the model to load. Model may be also provided with code keyword argument.

    		 None
    init dict or list of dicts

    Specifies initial values for parameters. It can be either a dictionary providing initial values for parameters used as keys, or a list of dictionaries providing initial values separately for each chain. If omitted, initial values will be generated automatically.

    Additionally this option allows to configure random number generators using following special keys:

    • '.RNG.name' str, name of random number generator
    • '.RNG.seed' int, seed for random number generator
    • '.RNG.state' array, may be specified instead of seed, shape of array depends on particular generator used
    		 None
    data dict

    Dictionary with observed nodes in the model. Keys are variable names and values should be convertible to numpy arrays with shape compatible with one used in the model.

    The numpy.ma.MaskedArray can be used to provide data where some of observations are missing.

    		 None
    generate_data bool

    If true, data block in the model is used to generate data.

    		 True
    chains int, 4 by default

    A positive number specifying number of parallel chains.

    		 4
    adapt int, 1000 by default

    An integer specifying number of adaptations steps.

    		 1000
    encoding str, 'utf-8' by default

    When model code is provided as a string, this specifies its encoding.

    		 'utf-8'
    progress_bar bool

    If true, enables the progress bar.

    		 True
    threads

    A positive integer specifying number of threads used to sample from model. Using more than one thread is experimental functionality.

    		 1
    chains_per_thread

    A positive integer specifying a maximum number of chains sampled in a single thread. Takes effect only when using more than one thread.

    		 1
    seed int

    Random seed for reproducible sampling. Deterministically derives per-chain RNG names and seeds using numpy.random.SeedSequence. Mutually exclusive with providing .RNG.name or .RNG.seed in init.

    		 None

    update(iterations) ¶

    Update the model for a given number of iterations.

    Runs the MCMC sampler without recording monitored values. This is typically used for burn-in.

    Parameters:

    Name Type Description Default
    iterations int

    A positive integer specifying the number of iterations to run.

    		 required

    adapt(iterations) ¶

    Run adaptation steps to maximize sampler efficiency.

    During adaptation, JAGS tunes its samplers to improve mixing. If the model does not require adaptation, this method returns immediately.

    Parameters:

    Name Type Description Default
    iterations int

    A positive integer specifying the number of adaptation steps.

    		 required

    Returns:

    Type Description
    bool

    True if the achieved performance is close to the theoretical optimum for all samplers.

    iter_sample(iterations=100000, chunk_size=1000, vars=None, thin=1, monitor_type='trace') ¶

    Yield sampling state after each chunk of iterations.

    A generator that samples chunk_size iterations at a time, merging results incrementally. Useful for convergence monitoring, live diagnostics, and checkpointing.

    Parameters:

    Name Type Description Default
    iterations int

    Maximum total iterations to sample.

    		 100000
    chunk_size int

    Number of iterations per chunk.

    		 1000
    vars list of str

    Variables to monitor. Defaults to all model variables.

    		 None
    thin int

    Thinning interval.

    		 1
    monitor_type str

    Monitor type (default 'trace').

    		 'trace'

    Yields:

    Type Description
    SamplingState

    Accumulated samples and diagnostics after each chunk.

    Examples:

    >>> for state in model.iter_sample(iterations=50000, chunk_size=5000):
...     if max(state.rhat.values()) < 1.01:
...         break
>>> final_samples = state.samples

    		 1
    monitor_type str

    		 'trace'

    Returns:

    Type Description

    sample(iterations, vars=None, thin=1, monitor_type='trace', warn_convergence=False) ¶

    Creates monitors for given variables, runs the model for provided number of iterations and returns monitored samples.

    Parameters:

    Name Type Description Default
    iterations int

    A positive integer specifying number of iterations.

    		 required
    vars list of str

    A list of variables to monitor.

    		 None
    thin int

    A positive integer specifying thinning interval.

    		 1
    warn_convergence bool

    If True, print warnings after sampling if any variable has R-hat > 1.01 or ESS < 100 per chain.

    		 False

    Returns:

    Type Description
    dict

    Sampled values of monitored variables as a dictionary where keys are variable names and values are numpy arrays with shape: (dim_1, dim_n, iterations, chains). dim_1, ..., dim_n describe the shape of variable in JAGS model.

    SamplingState ¶

    State yielded by :meth:Model.iter_sample at each chunk boundary.

    Attributes:

    Name Type Description
    samples dict[str, ndarray]

    Accumulated samples so far, with shape (*variable_dims, iterations, chains).
    iteration int

    Total number of iterations sampled so far.

    ess property ¶

    Per-variable effective sample size (computed lazily).

    rhat property ¶

    Per-variable R-hat statistic (computed lazily).

    chain_divergence property ¶

    Pairwise chain divergence matrix via energy distance.

    Requires the divergence package. Returns None if it is not installed. Computed lazily on first access.

    Returns:

    Type Description
    dict[str, ndarray] or None

    Mapping of variable names to (n_chains, n_chains) pairwise energy distance matrices, or None if divergence is not available.
